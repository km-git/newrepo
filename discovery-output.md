## 3. 4-Tier Model Routing Pattern

```
TIER 1 — FRONTIER ARCHITECT (5-10 calls/month total)
├── Fable 5: hardest architect, 1000+ line refactors, complex security, NDIS/compliance schemas
├── Opus 4.8: standard architect, schema design, parallel subagents, security review
└── GPT-5.6 Sol: UI architecture, voice agents, function calling, multi-agent orchestration

TIER 2 — BALANCED HEAVY (20-30% of work)
├── Sonnet 5: daily coding default, agent processing, code review, complex features
├── GPT-5.6 Terra: OpenAI alternative for standard builds
└── Gemini 3.1 Pro: 1M+ context docs, multimodal, AU-specific research

TIER 3 — CHEAP WORKHORSE (60-70% of work)
├── Haiku 4.5: Cmd+K inline edits, docstrings, simple refactors, prompt tuning
├── GPT-5.6 Luna: boilerplate, config files, simple completions
└── Gemini 3.5 Flash / Grok 4.5: trivial edits, background batch tasks

TIER 4 — CURSOR BUILT-IN
├── Auto mode: routine ops when unsure which model
├── Max Mode: big-context codebase-wide refactors
├── Composer 1: multi-file scaffold and batched refactors
└── BugBot: debugging (3x faster, 22% cheaper)

RULES:
- NEVER Tier 1 for workhorse edits
- NEVER Tier 3 for architect decisions
- 1-2 Fable 5 calls per MVP (initial plan only)
- 5-10 Opus 4.8 / Sol calls per MVP (feature design)
- 30-50 Haiku / Luna calls per MVP (inline edits)
```

```
  → Scaffold: Composer 1

  → Capture failure modes
       ↓
VERIFY (Tier 2: Sonnet 5)
  → ISC met? YES → Finalize | NO → back to BUILD
  → 1-3K tokens
       ↓
FINALIZE
  → Document outcome, update portfolio
  → Success → continue | Failure → cut bet within 7 days
       ↓

  → Friday portfolio review
  → 2.5% rule: expect 1-2 winners from 50-55 bets
```

## 7. Cursor Pro Power User Tips

1. **One Fable 5 call per MVP, not per feature.** Use your single monthly Fable call for the overall architect plan (schema + ISC + file structure). Everything else routes to Opus 4.8 or Sonnet 5. This alone saves 80%+ on frontier tokens across a 50-bet portfolio.

2. **Cmd+K is your default edit tool, not chat.** Open the file, select the code, Cmd+K with Haiku 4.5. Chat is for planning and multi-file work only. Operators who switch to 70% Cmd+K typically cut token spend in half within a week.

3. **Composer for day-1 scaffold, then never again until next MVP.** One Composer session to generate Next.js + Supabase + Stripe + auth boilerplate. After that, all changes via Cmd+K (Haiku) or targeted Sonnet 5 chat. Don't re-scaffold mid-build.

4. **Pre-load context with .cursorrules + @file references.** Put stack, conventions, and model routing in `.cursorrules` (free). Reference only the 2-3 files relevant to the current task with @ mentions. Never dump the whole codebase into context.

5. **Friday portfolio review with 7-day cut rule.** Every Friday: which bets have paying customers or clear signal? Kill anything with zero traction after 7 days post-launch. Preserved tokens from cut bets fund the next experiment. Target: 1-2 winners from 50-55 bets (2.5% rule).
