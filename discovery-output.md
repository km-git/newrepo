## 3. Cursor Pro Model Routing (Two-Pillar)

**All models below are selected in the Cursor IDE model picker. No external APIs.**

```
PILLAR 1 — WORKHORSE (60-70% of calls, execution volume)
├── Haiku 4.5 — Cmd+K edits, implementation, refactors, prompt tuning
├── GPT-5.6 Luna — boilerplate, config, types
├── Gemini 3.5 Flash — trivial edits, formatting
├── Grok 4.5 / DeepSeek — cheap code tasks
├── Opus 4.5/4.6/4.7 — FREE in Cursor Pro, workhorse escalation
├── BugBot — debugging (Cursor built-in)
└── Tab autocomplete — free

PILLAR 2 — HIGH-END ARCHITECT (5-15 calls/MVP, planning + decisions)
├── Opus 4.8 — standard architect, schema, design, fork decisions
├── Fable 5 — hardest architect, compliance, security, complex refactors
├── GPT-5.6 Sol — UI architecture, voice agents, multi-agent design
└── Sonnet 5 — complex logic, code review, ISC verification

CURSOR BUILT-IN
├── Auto mode — routine ops, Cursor picks cheapest adequate model
├── Composer 1 — multi-file scaffold, one session per MVP
├── Max Mode — big-context refactors (1M tokens)
└── BugBot — debugging

RULES:
- Select high-end model in Cursor for PLAN / DECIDE / VERIFY phases
- Switch to Haiku 4.5 in Cursor for BUILD / EDIT phases
- Cmd+K + Haiku for 70%+ of code changes
- Never use frontier models for docstrings or boilerplate
- Never use Haiku for architecture fork decisions
- All usage counts against A$20/mo Cursor Pro pool — route smartly
```

```

  → ISC, schema, file plan, dependencies
       ↓
SCAFFOLD — Composer (Cursor built-in, Auto mode)
  → Next.js + Supabase + Stripe boilerplate, one session
       ↓
BUILD — switch to Haiku 4.5 in Cursor (10-25K tokens)
  → Cmd+K for 70%+ of edits
  → Luna/Flash for boilerplate
       ↓
DECIDE — switch back to Opus 4.8 if blocked (0-2 calls)
  → Architecture fork, security, cut/continue
       ↓
TEST — no AI
       ↓
VERIFY — select Sonnet 5 in Cursor (1-3K tokens)
  → ISC met? YES → Finalize | NO → Haiku fix
       ↓
FINALIZE — document, cut or continue (max 5 cycles)
```
