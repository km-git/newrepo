## 3. 4-Tier Routing Pattern

| Tier | Models | When | Budget |
|------|--------|------|--------|
| **T1 Frontier** | Fable 5, Opus 4.8, GPT-5.6 Sol | Hardest architect, schema, security, UI/tool-calling design | 5–10 Fable + ~20–30 Opus/Sol calls/mo |
| **T2 Balanced** | Sonnet 5, Terra, Gemini 3.1 Pro, Opus 4.5–4.7 | Daily coding, review, long docs, multimodal | ~20–30% of work; Sonnet 5 = default |
| **T3 Workhorse** | Haiku 4.5, Luna, Flash, Grok 4.5, DeepSeek | Cmd+K edits, boilerplate, docstrings, simple fixes | 60–70% of work |
| **T4 Built-in** | Auto, Max, Composer 1, BugBot | Unsure / big context / multi-file / debug | Routine + scaffold + debug |

**Decision rule:** Never T1 for workhorse. Never T3 for architect. Sonnet 5 for ~90% of daily coding. Auto when unsure. Max for 1M-context refactors. Composer for multi-file. BugBot for bugs.

**Per-MVP call mix:** 1–2 Fable (or Opus if Fable unavailable) · 5–10 Opus/Sol · 1–3 Composer · 30–50 Haiku/Luna Cmd+K · 20–40 Luna/Flash boilerplate.

## 4. Token-Saving Cheat Sheet

| Lever | Do this | Why |
|-------|---------|-----|
| Cmd+K first | ~70% of edits inline | Doesn’t bloat chat context |
| Cache sessions | Keep one long chat for a feature | Cursor reuses cached context |
| `.cursorrules` | Project root instructions | Free, always-on constraints |
| Snippets | Stripe/auth/Prisma snippets | Saves 200+ tokens vs regen |
| Tab autocomplete | Aggressive for boilerplate | Near-free completions |
| Tier routing | Cheapest adequate model | 70–90% vs always-frontier |
| Small files | Point at exact files in prompts | Avoid full-repo dumps |
| ISC first | 1–2 sentence success criteria | Cuts rework loops |
| Composer batch | Multi-file in one go | Fewer chat round-trips |
| BugBot for bugs | Don’t burn Sonnet on stack traces | 3× faster / 22% cheaper |

## 5. Continuous Testing Loop (max 5 cycles)

```
PLAN (T1: Fable 5 or Opus 4.8)
  → ISC in 1–2 sentences + schema + file plan
BUILD (T2–T3: Sonnet 5 + Haiku 4.5; Composer scaffold; Sol for complex UI)
TEST (mostly human)
  → unit + smoke; capture failure modes
VERIFY (T2: Sonnet 5)
  → ISC met? YES → FINALIZE | NO → BUILD (cheap fix vs frontier if architectural)
FINALIZE
  → document outcome; portfolio update; 7-day cut if failing
LOOP
  → max 3–5 cycles/MVP; Friday portfolio review
```

**ISC example (QuoteGuard):** “User pastes job notes → SMS-ready quote with labour/materials + ≥25% margin warning in <60s.”
